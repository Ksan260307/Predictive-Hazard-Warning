# 先読み危険予測 v2.0.0

カメラ映像と位置情報から、衝突・飛び出しなどの危険を**事前に**予測し、
「安全 / 注意 / 危険」の3段階で知らせるアプリです。**スマートフォンの横持ち**を想定しています。

「いま危ないか」ではなく「**この先どうなりうるか(未来の分布)**」で判断します。

## 主な機能

- **移動モード**: 徒歩 / 自転車 / 車。速い乗り物ほど遠くまで・早めに警告します
- **AI物体認識 (任意)**: YOLOv8モデルを導入すると、止まっている人・車・自転車なども
  認識し、警告に「歩行者が接近しています」など**物の名前**が出ます
  (モデルが無い環境では従来の動き検出で動きます。導入方法は下記)
- **移動対応の物体検出**: 自分の移動によるカメラ全体の動きを推定して打ち消し、
  本当に動いている物だけを検出します
- **検出の安定化**: 数フレーム連続で見えた物だけを警戒の対象にし、
  光のちらつき等による「1フレームだけのノイズ」を捨てます
- **飛び出し予測**: 建物の角・塀・駐車車両などの「死角」を映像から検出し、
  そこから人や車が飛び出してくる可能性を未来の分布に織り込みます。
  AI物体認識の導入時は「止まっている車の陰」も確かな根拠つきの死角として扱います
- **省電力**: 何も起きていない時間が続くと分析を自動で間引き、電池と発熱を抑えます。
  何かが見えた瞬間・危険度が上がった瞬間は、すぐ毎フレーム分析に戻ります
- **位置情報との統合**: GPSから速度・旋回を求め、OpenStreetMapの道路データから
  実際の交差点への接近を判定し、画像分析と合わせて総合的に危険度を判断します
  (APIキー不要。圏外でもGPSだけで動きます)
- **学習の安全弁**: 報告がまだ少ないうちは学習を控えめに効かせ、
  数件の報告だけで予測が大きく振れないようにします
- **走行ログ (任意)**: 危険度・レベル・理由・報告を時系列で記録し、
  誤報が出た場面の見直しや感度調整に使えます。
  「誤報を報告」「見逃しを報告」した瞬間は、直前数秒の**場面の画像**も保存され、
  ログから「その時、実際に何が映っていたか」を見返せます
- **音声読み上げ (任意)**: 警告の内容 (「危険。歩行者が接近しています」など) を
  声で読み上げます。運転中に画面を見なくても状況が分かります
- **学習**: 「誤報を報告」「見逃しを報告」ボタンからの指摘を取り込み、
  同じ状況での予測の強さを自動調整します(学習結果は保存され次回も有効)
- **映像HUD**: 危険域の網かけ・検出物体の枠と進行方向の矢印・死角マーカー・
  危険度バーを映像に重ねて表示します
- **警告履歴**: 直近の警告を時刻・理由つきで一覧表示します
- **一時停止**: 監視を止めて映像だけ見ることができます
- **デモ映像**: カメラの無いPCでも動作を確認できます (カメラ未接続時は自動で切替、
  接続を検出すると自動復帰。設定で常時デモにもできます)

## 動かし方 (PCで試す)

```
pip install -r requirements.txt
python main.py
```

カメラ付きのPCならそのまま動きます。カメラが無くてもデモ映像で動きます。
画面はスマートフォンの横持ちと同じ比率です。

### AI物体認識を有効にする (任意)

```
pip install ultralytics
python tools/download_model.py
```

`data/yolov8n.onnx` が配置され、次回起動からAIによる物体認識が自動で有効になります。
モデルが無い環境・「AIで物体を認識」をオフにした場合は、従来の動き検出で動きます。

## テスト

```
python -m pytest tests -v
```

全モジュールの単体テスト 360件超 (正常系・異常系・境界値・ランダムテスト) を収録。

## スマートフォン (Android) への導入

`buildozer.spec` を同梱しています。Linux (または WSL) 上で:

```
pip install buildozer
buildozer android debug
```

> 注意: Android 実機ではカメラ取得 (`cv2.VideoCapture`) が機種により動作しない
> ことがあります。その場合は `camera4kivy` 等への差し替えが必要です。
> 分析エンジン (`app/` 以下) はそのまま利用できます。
> GPS は `plyer` 経由で取得します (位置情報の権限が必要)。

## 画面の見方

| 表示 | 意味 |
|------|------|
| 安全 (緑) | 周囲に差し迫った危険はありません |
| 注意 (黄) | 注意が必要な状況です |
| 危険 (赤) | 衝突の恐れがあります (音でも通知) |

- 警告時は「接近する物体があります」「見通しの悪い箇所があります」
  「この先に交差点があります」など、**理由**も表示されます
- 映像上には、検出した物体 (橙の枠) と死角の候補 (黄の縦線) が表示されます
- 下部には危険度・内部状態 (探索/適応/安定)・速度が表示されます

## 設定

| 設定 | 意味 |
|------|------|
| モード (監視画面) | 徒歩 / 自転車 / 車 |
| カメラ番号 | 使用するカメラの切り替え |
| デモ映像を使う | カメラの代わりに合成映像で動かす |
| 検出感度 | 高いほど小さな動きも拾う |
| 予測の先読みの長さ | 何歩先まで予測するか (モードで自動調整) |
| 予測の本数 | 未来を何通り考えるか |
| 予測の更新の速さ | 高いほど最新の状況をすぐ反映 |
| 警戒を強める危険度 | この危険度を超えたら警戒を強める |
| 危険域とみなす画面位置 | 画面のどこから下を衝突域とするか (モードで自動調整) |
| AIで物体を認識 | モデル導入時、人・車などをAIで認識 (無ければ動き検出) |
| 静かな時は分析を減らす | 何も起きていない間は分析を間引いて電池を節約 |
| 移動による映像の揺れを補正する | 移動しながら使う場合はオン |
| 位置情報(GPS)を使う | 速度・旋回・交差点判定に使用 |
| 報告から学習する | 誤報・見逃しの報告を予測に反映 |
| 検出する物の最小の大きさ | これより小さい物はノイズとして無視 |
| 走行ログを保存する | 危険度や報告を triplog.jsonl に記録。報告時は場面の画像も trip_snapshots/ に保存 |
| 警告の内容を声で読み上げる | 警告時に内容を音声で通知 (対応環境のみ) |

設定は `settei.json`、学習結果は `learned.json` に保存されます。
どちらも壊れていた場合は初期値で安全に起動します。

## 仕組み

```
カメラ画像 ──┬─ ① 物の検出                                … objectdetect.py (AI認識。モデル導入時)
             │      └ モデルが無ければ動き検出              … detector.py
             │      └ 連続して見えた物だけを通す            … tracker.py
             └─ ② 死角・見通しの検出                       … scene.py
GPS ────────── ③ 速度・旋回・交差点への接近               … roadinfo.py
                 ↓
④ 見える物の未来 + 死角からの飛び出し(見えない未来) を生成 … future.py
⑤ 古い予測を薄めて新しい予測を混ぜる                       … future.py
⑥ 危険度・不確かさ・余裕を計算                             … risk.py
⑦ 地図情報の危険と合成し、警戒の強さを決定                 … control.py
⑧ 注意レベルを選択                                         … control.py (softmax による行動選択)
⑨ 言葉・色・音で通知                                       … alert.py
⑩ ユーザーの報告で予測の強さを学習                         … learning.py
```

危険予測エンジンの特徴:

- **見えない未来も扱う**: 死角からの飛び出しという「まだ見えていない未来」を
  分布に混ぜ、観測する前から警戒に使う
- **予兆駆動**: 物体が衝突域に「到達する前」に未来の分布が先に危険を示す
- **リスクの連続的減衰**: 物や死角が消えても予測は徐々に薄れる
- **最小行動量の保証**: 警戒が強い時は必ず通知する
- **学習機構**: ユーザーの指摘で予測の重みをオンライン更新
- **計算量制御**: 未来の保持数の上限と低確率の切り捨て

なお、学習が調整するのは飛び出し・交差点などの**推測に基づく予測**のみで、
実際に見えている物体の衝突予測は学習で弱められない設計です(安全側に倒す)。

## ファイル構成

```
main.py            画面 (Kivy)。横持ち2カラムUI・カメラ・GPS・通知
app/
  settings.py      設定の読み書き。不正値の自動補正
  modes.py         徒歩/自転車/車のパラメータ
  detector.py      動く物の検出 (カメラ自身の動きを打ち消し)・分析用縮小
  objectdetect.py  AIモデル(YOLOv8)による人・車などの認識 (モデル導入時)
  tracker.py       検出結果の時間方向のならし (ノイズ除去・速度の平滑化)
  scene.py         死角・見通しの検出 (風景の縦境界 + 止まっている車の陰)
  power.py         省電力のための分析の間引き
  roadinfo.py      GPS履歴の分析と OpenStreetMap 連携 (交差点の取得)
  triplog.py       走行ログの記録 (JSONL形式)
  snapshot.py      報告があった場面の画像の保存
  future.py        未来の生成・保持。飛び出しの仮想未来も生成
  risk.py          危険度・不確かさ・余裕の計算
  control.py       警戒の強さの決定 (制御・位相・行動選択)
  learning.py      報告からの学習 (保存・復元つき)
  alert.py         通知の言葉・色・音
  hud.py           映像への描き込み (危険域・矢印・死角マーカー・危険度バー)
  demo.py          デモ映像の合成
  watcher.py       上記を1つにつないだ入り口
tests/             単体テスト
tools/make_icon.py アイコン・起動画面の生成スクリプト
tools/download_model.py  AI物体認識モデルの導入スクリプト
data/              アイコン (icon.png)・起動画面 (presplash.png)・
                   AIモデル (yolov8n.onnx。導入時のみ)
settei.json        保存された設定 / learned.json 学習結果 (初回起動時に作成)
buildozer.spec     Android用の設定 (横画面・アイコン・権限)
```
